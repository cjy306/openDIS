## 1. 研究背景与动机

### 1.1 离散位错动力学（DDD）简介

离散位错动力学（Discrete Dislocation Dynamics, DDD）是研究金属塑性的重要介观尺度方法，能够在原子模拟难以触及的长度和时间尺度上对大量位错集合的运动与相互作用进行模拟。在DDD中：

- 位错微观结构被离散为由**位错节点**（nodes）通过线段（segments）相互连接的网络
- 每个时间步分两个主要阶段：
  1. **时间积分**：对所有节点位置进行时间推进
  2. **拓扑操作**：处理相交线段（如结点形成、annihilation等）

### 1.2 核心计算瓶颈

DDD模拟的最大计算挑战在于**节点力的计算**：

- 位错产生长程应力场，衰减规律为 $1/R$
- 节点力 $\mathbf{F}$ 需对所有线段对（segment pairs）进行积分：

$$\mathbf{f}_{PK} = (\boldsymbol{\sigma}_{tot} \cdot \mathbf{b}) \times \mathbf{t}$$

其中 $\mathbf{b}$ 为Burgers矢量，$\mathbf{t}$ 为位错线单位切向量。

- 暴力计算复杂度为 $\mathcal{O}(N_{seg}^2)$，随位错增殖急剧增长
- 随着应变增大，位错密度不断增加，计算量雪崩式上涨

### 1.3 时间积分刚性问题

传统全局时间积分器（如Heun、Runge–Kutta–Fehlberg, RKF）存在严重缺陷：

- 时间步长必须全局统一，受**最不稳定节点**（最小步长需求）制约
- 系统固有刚性 + 短程强非线性 → 时间步极小
- 随位错网络细化，可达到的应变量太低、应变率太高，距实验条件差几个数量级

## 2. 核心方法：子循环时间积分

### 2.1 基本思想

**子循环（Subcycling）时间积分器**（由Sills et al. 2016提出）的核心思想：

> 将相互作用力按**作用距离**分组，对不同组采用**独立步长**分别积分，从而将快速（短程、不稳定）模式从系统中隔离出来单独处理。

### 2.2 五组力分类方案（Force-based Subcycling）

| 组号 | 作用范围 | 行为特征 | 积分步长 |
|------|----------|----------|----------|
| Group 0 | 长程（Long-range）| 缓慢变化，行为良好 | 最大（全局步长 $\Delta t$） |
| Group 1 | 自作用 + 应力/FMM | 含核能、施加应力、远场FMM | 次之 |
| Group 2–4 | 短程（Short-range）| 快速变化，刚性强 | 最小（多次子循环） |

**积分流程：**
1. 先用 Group 0（长程力）推进所有节点，确定全局步长 $\Delta t$
2. 依次对 Group 1→4 进行子循环积分，直至每组在时间上追上 Group 0
3. 对于难以收敛的"麻烦节点"，将其所有相关相互作用转移至 Group 4

**性能提升：** 与简单Heun积分器相比，计算时间提升约**2个数量级**。

### 2.3 MPI并行的可扩展性瓶颈

子循环方案虽然大幅减少了力计算量，但在MPI并行时暴露出新问题：

- 每个子循环之间都需要**节点位置通信**
- 力计算量降低后，通信开销的**相对比重**急剧上升
- 导致并行可扩展性严重退化

→ 这正是引入GPU加速的根本动机。

## 3. GPU加速实现

### 3.1 整体设计哲学

GPU加速特别适合子循环方案，原因有二：

1. **消除通信开销**：整个子循环过程在单个GPU内部完成，无需GPU–CPU间频繁通信
2. **自由度恒定**：时间积分过程中不进行拓扑操作，节点数和连接关系固定，内存传输极少

**数据流设计：**
```
CPU → GPU：初始节点位置 r_t + 线段列表 + 线段对列表（一次性传输）
GPU内部：完整子循环积分（Group 0 → 4，全程在GPU）
GPU → CPU：更新后的节点位置、力、速度 r_{t+Δt}（一次性回传）
```

### 3.2 各核函数（Kernel）详细说明

#### 3.2.1 分组核（Groups Assignment Kernel）

- 在进入子循环前，于**CPU**上构建完整线段对列表（无算术操作，速度快）
- 将列表传至GPU，**并行计算每对线段的最近距离**
- 根据分组半径 $r_{g_i}$ 将每个线段对分配至对应组
- 大多数线段对留在 Group 0；少量被标记为 Group 1–4

#### 3.2.2 RKF积分核（RKF Kernel）

- 每个子循环使用**五阶Runge–Kutta–Fehlberg（RKF）积分器**
- 需要6次中间力/速度计算逐步构建解
- 关键辅助数组：
  - 大小 $3 \times 6N_{node}$：存储中间节点速度
  - 大小 $N_{node}$：实现"前向进展检查"（forward progress check），防止节点在平衡位置附近振荡

#### 3.2.3 力计算核（Forces Kernel）

类比有限元（FEM）的**单元组装流程**：

1. **单元力计算**：每个线程处理一个线段对（4节点单元），使用**非奇异相互作用力公式**（Cai et al. 2006），涉及 >1500 次算术运算
2. **结果暂存**：将单元力存入大小 $3 \times 4N_{pair}$ 的全局单元数组（避免竞争条件 race condition）
3. **节点组装**：通过归约映射（reduction map）将单元力累加至各节点总力

归约映射由两个数组组成：
- 位置数组（大小 $4N_{pair}$）：记录每个节点需要取哪些单元力
- 辅助索引数组（$N_{node}$ 个2元组）：每个节点在位置数组中的范围

**Group 1专用核**：处理自作用力 + 施加应力 + FMM远场力（后者在进入子循环前于CPU预计算，子循环中假设不变）

#### 3.2.4 速度计算核（Velocity Kernel）

**线性标量迁移率律（FCC典型）：**

节点力—速度关系简化为矩阵-向量乘法：

$$\mathbf{V}_i = \mathbf{M}_i \mathbf{F}_i$$

其中迁移率矩阵分解为：

$$\mathbf{M}_i = \left(\prod_c \mathbf{P}_c\right) \mathbf{B}_i^{-1}$$

- $\mathbf{P}_c$：滑移面约束投影算子（预计算，子循环中不变）
- $\mathbf{B}_i$：节点阻尼矩阵（依赖线段长度，需每步更新）

**非线性标量迁移率律（扩展）：**

针对非线性力—速度关系，提出**切线速度近似方案**：

- 在总力 $F_i^t$ 处线性化，得到有效阻尼系数 $B^*$
- 定义有效反力（back-force）：$\mathbf{F}_i^* = F_i^t - f^* \hat{\mathbf{F}}_i^t$
- 子循环中使用 $B^*$ + 将 $\mathbf{F}_i^*$ 加入 Group 1
- 无需额外力计算，只需额外两个数组（大小 $N_{node}$ 和 $3N_{node}$）

### 3.3 其他GPU加速模块

#### 节点分裂（Node Splitting）

- 用于判断碰撞后形成的多臂节点是否应分离
- 对每种试探配置，用GPU加速计算分离节点的线段对相互作用力
- 采用顺序实现（每种配置独立传输线段对列表）

#### 部分力更新（Partial Forces Update）

- 拓扑操作后，对发生变化的节点重新计算力
- GPU加速线段对相互作用力计算

#### 快速多极方法（FMM）加速

- FMM最耗时的阶段：计算各FMM单元的多极矩（Taylor展开系数）
- 每层需要 $189 N_c p^4$ 次运算（$p$：展开阶数，$N_c$：单元数，189：邻居数）
- GPU并行计算每层的 $189N_c$ 个Taylor系数贡献，使用CUDA原子操作进行累加

## 4. 追溯碰撞算法（Retroactive Collision Algorithm）

### 4.1 问题背景

由于子循环方案的全局步长 $\Delta t$ 可能很大，需要一个鲁棒的碰撞检测算法避免漏检。

### 4.2 算法框架

考虑时间区间 $[t, t+\Delta t]$ 内两段移动的位错线段：
- 线段1：$[\mathbf{r}_1^t, \mathbf{r}_2^t]$ → $[\mathbf{r}_1^{t+\Delta t}, \mathbf{r}_2^{t+\Delta t}]$
- 线段2：$[\mathbf{r}_3^t, \mathbf{r}_4^t]$ → $[\mathbf{r}_3^{t+\Delta t}, \mathbf{r}_4^{t+\Delta t}]$

两段上的移动点（假设节点匀速运动）：

$$\mathbf{P}(\tau) = (1-u)\mathbf{r}_1(\tau) + u\mathbf{r}_2(\tau)$$
$$\mathbf{Q}(\tau) = (1-v)\mathbf{r}_3(\tau) + v\mathbf{r}_4(\tau)$$

其中 $w = (\tau - t)/\Delta t \in [0,1]$，$u, v \in [0,1]$。

### 4.3 最小化问题

碰撞检测等价于在单位立方体 $[0,1]^3$ 内最小化距离平方函数：

$$F(u, v, w) = \|\mathbf{P}(\tau) - \mathbf{Q}(\tau)\|^2$$

**求解策略（按计算成本升序）：**

| 情形 | 变量约束 | 求解方法 |
|------|----------|----------|
| 8个顶点 | $u,v,w \in \{0,1\}$ | 直接计算两点距离 |
| 12条棱 | 一维约束 | 解一元方程 $dF/d(\cdot)=0$ |
| 6个面（$w$固定） | 二维线性系统 | 解线性方程组 $\partial F/\partial u = \partial F/\partial v = 0$ |
| 6个面（$u$或$v$固定）| 二维非线性 | Newton迭代 |
| 内部通解 | 无约束 | Newton迭代（三维） |

一旦最小距离满足预设碰撞距离，立即退出并声明发生碰撞。

**球形预过滤：** 对每对移动线段预先用包围球过滤，排除绝大多数不可能碰撞的线段对，大幅减少详细计算的数量。

## 5. 性能测试与结果

### 5.1 基准测试1：FCC铜，大尺寸模拟

**参数设置：**

| 参数 | 值 |
|------|----|
| 模拟盒尺寸 | $(15\ \mu\text{m})^3$ |
| 初始位错密度 | $\rho_0 = 0.7 \times 10^{12}\ \text{m}^{-2}$ |
| 加载方向 | [001] 单轴压缩 |
| 应变率 | $\dot{\varepsilon} = 10^3\ \text{s}^{-1}$ |
| 初始线段数 | ~2000 |
| 迁移率律 | 线性标量，$B = 15.6\ \text{Pa·s·m}$ |
| FMM网格 | $4^3$ 单元（粗，突出集成效果） |
| 硬件对比 | 1× Intel Xeon X5650 (2.67 GHz) vs. 1× NVidia GTX Titan X |

**结果：**

| 版本 | 达到 $\gamma = 0.8\%$ 所需时间 |
|------|-------------------------------|
| CPU（子循环，已比简单Heun快~100×） | ~1200 CPU小时（50天） |
| GPU加速（本工作） | ~32小时 |
| **GPU加速比** | **≈37×** |

在 $\gamma = 0.8\%$ 时，位错密度增至 $\rho = 1.7 \times 10^{12}\ \text{m}^{-2}$，约10,000条线段。GPU版本与CPU版本给出一致的应力-应变响应。

**各阶段加速分析：**

| 阶段 | CPU版本占比 | GPU加速倍数 | GPU版本占比 |
|------|------------|------------|------------|
| 时间积分 | ~98% | 50× | ~50% |
| 节点分裂 + 部分力更新 | 小 | — | 显著（不可忽略） |
| 线段对距离计算（分组） | 小 | GPU显著提升 | — |

**关键结论：** 仅加速时间积分不够，必须同时加速节点分裂、部分力更新等含力计算的阶段，才能保留整体加速比。

### 5.2 基准测试2：不同应变率下的性能（$8^3$ FMM单元）

| 应变率 $\dot{\varepsilon}$ | 达到 $\gamma = 1\%$ 所需时间（1 CPU + 1 GPU） |
|---------------------------|----------------------------------------------|
| $10^4\ \text{s}^{-1}$ | **3小时** |
| $10^3\ \text{s}^{-1}$ | **1天** |
| $10^2\ \text{s}^{-1}$ | **15天** |

### 5.3 基准测试3：FCC铝，高密度大应变模拟

**参数：**

| 参数 | 值 |
|------|----|
| 模拟盒尺寸 | $(1.8\ \mu\text{m})^3$ |
| 初始位错密度 | $\rho_0 = 0.4 \times 10^{14}\ \text{m}^{-2}$ |
| 最大线段离散尺寸 | $50b$（$b = 0.286\ \text{nm}$，Al的Burgers矢量） |
| 初始线段数 | ~20,000 |
| 应变率 | $\dot{\varepsilon} = 10^5\ \text{s}^{-1}$ |
| FMM网格 | 从 $8^3$ 动态增至 $32^3$ |
| 目标应变 | $\gamma = 10\%$ |

**结果：** 
- 使用 **1 CPU + 1 GPU**，仅用 **10天** 完成 $\gamma = 10\%$ 的大应变模拟
- 终态：位错密度 $\rho = 2 \times 10^{14}\ \text{m}^{-2}$，~**130,000条线段**

### 5.4 GPU内存容量限制

GPU设备内存主要被**线段对数据**占用（每对约150字节），对于12 GB显存设备（GTX Titan X / Tesla K40）：

$$N_{pair} = \frac{27}{N_{FMM}} \cdot \frac{N_{seg}^2}{2}$$

| FMM单元数 $N_{FMM}$ | 可处理最大线段数 $N_{seg}$ |
|--------------------|--------------------------|
| $4^3 = 64$ | ~20,000 |
| $32^3 = 32768$ | ~450,000 |

## 6. 结论与展望

### 6.1 主要贡献

1. **方法创新**：将子循环时间积分与GPU大规模并行计算首次有机结合
2. **性能突破**：
   - 相比简单积分器（Heun）：加速约**100×**（来自子循环）
   - 相比CPU版子循环：额外加速**30–40×**（来自GPU）
   - 综合提升：相比传统方法加速约**3000–4000×**
3. **规模可达**：单GPU即可完成大规模DDD模拟，实现实验相关应变量
4. **可扩展性**：解决了MPI方案中子循环通信瓶颈问题

### 6.2 局限性与未来方向

- 当前实现仅支持**单GPU**；多GPU实现需要仔细设计GPU间通信
- CPU上剩余阶段可用**OpenMP多线程**进一步加速
- 随着线段数增多，GPU显存容量成为限制，需要更大显存或分布式方案

## 7. 方法体系结构总览

```
DDD 全局时间步
│
├── [GPU] 子循环时间积分（~50%时间）
│   ├── 分组分配核（Group Assignment Kernel）
│   ├── Group 0：长程力 RKF积分 → 确定全局步长 Δt
│   ├── Group 1：自作用 + 施加应力 + FMM（预计算）→ 子循环
│   ├── Group 2–4：短程力 → 子循环（步长最小）
│   └── 速度核（线性/非线性迁移率律）
│
├── [GPU] 节点分裂（Node Splitting）
│   └── 每个试探配置 → GPU计算线段对力 → CPU判断耗散最大构型
│
├── [GPU] 部分力更新（Partial Forces Update）
│   └── 拓扑操作后对变化节点重算力
│
├── [GPU] FMM多极矩计算
│   └── 并行计算Taylor系数，CUDA原子操作累加
│
└── [CPU] 拓扑操作（碰撞检测/重网格/结点形成）
    └── 追溯碰撞算法（球形预过滤 → 特殊情形 → 内部通解）
```
